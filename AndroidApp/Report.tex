\section{Introducci\'on}
Este documento describe el desarrollo de una aplicaci\'on m\'ovil de \textbf{Control Acad\'emico} en Android (Kotlin) con backend en Firebase. El sistema gestiona materias, roles de usuario (Administrador, Profesor y Alumno), asistencia mediante c\'odigos QR y registro de calificaciones.

\section{Requisitos t\'ecnicos y tecnolog\'ias}
\begin{itemize}[leftmargin=*]
    \item \textbf{Lenguaje:} Kotlin.
    \item \textbf{Compatibilidad:} Android API 24+.
    \item \textbf{Autenticaci\'on:} Firebase Authentication (correo y contrase\~na).
    \item \textbf{Base de datos:} Firebase Cloud Firestore.
    \item \textbf{Almacenamiento local:} SharedPreferences (recordar sesi\'on).
    \item \textbf{QR:} ZXing (generaci\'on en Alumno y escaneo en Profesor).
\end{itemize}

\section{Arquitectura general}
La aplicaci\'on sigue una arquitectura sencilla por capas:
\begin{itemize}[leftmargin=*]
    \item \textbf{UI (Activities):} pantallas de Login/Registro, configuraci\'on de perfil, pantalla principal y paneles por rol.
    \item \textbf{Capa de datos (Repository):} operaciones de lectura/escritura en Firestore y mapeo a modelos.
    \item \textbf{Servicios externos:} Firebase Auth, Firestore y librer\'ia de QR.
\end{itemize}

\section{Modelo de datos (Firestore)}
\subsection*{Colecciones}
\begin{itemize}[leftmargin=*]
    \item \textbf{users}: perfil del usuario (uid, nombre, matr\'icula, correo, rol).
    \item \textbf{subjects}: materias (nombre, horario, profesor asignado, alumnos inscritos).
    \item \textbf{grades}: calificaciones por materia y alumno.
    \item \textbf{attendance}: registros de asistencia (materia, alumno, profesor, timestamp).
\end{itemize}

\subsection*{Accesos t\'ipicos}
\begin{itemize}[leftmargin=*]
    \item \textbf{Administrador:} consulta usuarios, cambia rol (solo a profesor/alumno), crea materias y asigna profesor/alumnos.
    \item \textbf{Profesor:} consulta sus materias, lista de alumnos por materia, registra calificaciones y asistencia.
    \item \textbf{Alumno:} consulta materias asignadas, genera QR para pase de lista y consulta calificaciones.
\end{itemize}

\section{Funcionalidades por rol}
\subsection{Administrador}
\begin{itemize}[leftmargin=*]
    \item Cambiar tipo de usuario (asignar como profesor o alumno).
    \item Crear nuevas materias.
    \item Asignar materias a profesor y alumnos.
\end{itemize}

\subsection{Profesor}
\begin{itemize}[leftmargin=*]
    \item Revisar horario de clases (materias asignadas).
    \item Asignar calificaciones a alumnos inscritos en la materia.
    \item Tomar lista de asistencia escaneando el c\'odigo QR del alumno.
\end{itemize}

\subsection{Alumno}
\begin{itemize}[leftmargin=*]
    \item Revisar horario de clases (materias asignadas).
    \item Generar un c\'odigo QR para el pase de lista en la materia actual.
    \item Ver calificaciones por materia.
\end{itemize}

\section{Flujo de uso (resumen)}
\begin{enumerate}[leftmargin=*]
    \item El usuario se registra o inicia sesi\'on con Firebase Authentication.
    \item Tras registrarse, completa su perfil (nombre/matr\'icula) y queda como alumno por defecto.
    \item La app carga el perfil desde Firestore y enruta al panel seg\'un el rol.
    \item El Administrador gestiona usuarios y materias.
    \item El Profesor registra calificaciones y asistencia (QR).
    \item El Alumno consulta materias/calificaciones y genera QR.
\end{enumerate}

\section{Conclusiones}
Se implement\'o una aplicaci\'on funcional de control acad\'emico que integra autenticaci\'on, almacenamiento en la nube, manejo de roles y funcionalidad de QR para asistencia. La arquitectura y el modelo de datos permiten extender la aplicaci\'on con nuevas pantallas o validaciones, manteniendo una separaci\'on clara entre UI y acceso a Firestore.
